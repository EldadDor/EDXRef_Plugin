Validates @WSHeader annotations defined on methods within consumer types.

This inspection checks methods inside classes/interfaces that are annotated with @WSConsumer and implement/extend WebserviceConsumer or PearlWebserviceConsumer.

It reports the following issues:

  • Error: Invalid Setter Default Value: If @WSHeader is placed on a setter method and the defaultValue attribute is present but empty (""), an error is reported. The attribute must have a non-empty value if specified on a setter.
  • Warning: Redundant Method Header: If a header name is specified via @WSHeader on a method, and the same header name is *also* defined at the type level (directly or via @WSHeaders), a warning is shown on the method's @WSHeader. The type-level definition takes precedence.

Example:


@WSConsumer(url = "...")
@WSHeaders({ @WSHeader(name = "X-Request-ID", defaultValue = "TypeID") })
public interface MyConsumer extends WebserviceConsumer {

    // OK: Header only on method, defaultValue not required/present
    @Property @WSHeader(name = "X-Custom-Data")
    void setCustomData(String data);

    // ERROR: defaultValue is present but empty on setter
    @Property @WSHeader(name = "X-Auth", defaultValue = "")
    void setAuthToken(String token);

    // WARN: Redundant, "X-Request-ID" defined on type
    @Property @WSHeader(name = "X-Request-ID", defaultValue = "MethodID")
    void setRequestId(String id);

    // OK: Header on getter, defaultValue allowed but not required to be non-empty
    @Property @WSHeader(name = "X-Response-Info", defaultValue = "")
    String getResponseInfo();
}
